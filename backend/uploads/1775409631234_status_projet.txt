Funzionalità già implementate e funzionanti
Autenticazione
/api/register → creazione account con password hash
/api/login → login e generazione JWT
Gestione file
/api/files/upload → upload file con scadenza (1–7 giorni) e opzionale password
/api/files/download/{id} → download file per utente autenticato
/api/files/my-files → lista storico file utente
/api/files/{id} → delete file (solo proprietario)
Sicurezza
JWT già implementato con JwtService + JwtAuthenticationFilter
Endpoint protetti tramite @AuthenticationPrincipal
Password hash sicuro (bcrypt)
Validazioni core
Expiration file tra 1–7 giorni
Controllo proprietà file prima di delete
File non trovato / scaduto → 404
